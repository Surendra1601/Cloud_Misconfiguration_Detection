Getting Started

Follow these steps to set up CloudLine on your computer using Docker Compose.

What You Need Before Starting

Simple Analogy Before baking a cake, you need an oven, ingredients, and a recipe. These are the "ingredients" you need before setting up CloudLine.
Tool Version What It Is How to Get It
Docker Any recent Runs services in containers (like mini virtual machines) docker --version
Docker Compose v2+ Orchestrates multiple containers docker compose version
Node.js 20+ Runs the frontend (dashboard) node --version
Git Any Downloads the CloudLine code git --version
💡
Not sure if you have these? Open your terminal (Command Prompt on Windows, Terminal on Mac/Linux) and type each command in the "How to Get It" column. If it shows a version number, you have it. If it says "command not found", you need to install it.

Setup

CloudLine runs entirely via Docker Compose. One command starts the backend, OPA policy engine, and a local DynamoDB database.

Download the CloudLine code

Open your terminal and run:

terminal git clone <repo-url> cd CloudLine

Start all services with Docker

This single command starts the backend server, the policy engine (OPA), and a local database:

terminal make docker-up

Wait about 30 seconds. You will see logs scrolling. When you see "Application startup complete", the backend is ready.

Start the dashboard (frontend)

Open a new terminal window and run:

terminal cd CloudLine/frontend npm install npm run dev

The dashboard will open at http://localhost:5173

Verify it works

Open another terminal and check the health endpoint:

terminal curl http://localhost:9710/health

You should see: {"status": "healthy"}

✅
That's it! The backend runs on port 9710, OPA on port 9720, and the database on port 9730. The dashboard runs on port 5173.

Environment Variables

These settings control how CloudLine behaves. Copy .env.example to .env and adjust as needed.

Variable Required? Default What It Does
AWS_REGION Yes us-east-1 Which AWS region to scan
AWS_ACCOUNT_ID Yes — Your 12-digit AWS account number
APP_ENV No development "development" or "production"
LOG_LEVEL No DEBUG How much detail to log (DEBUG, INFO, WARNING, ERROR)
OPA_MODE No http "cli" for local OPA, "http" for Docker sidecar
CORS_ORIGINS No http://localhost:5173 Which websites can access the API

Useful Commands

Command What It Does
make docker-up Start all services (backend + OPA + database)
make docker-down Stop all services
make docker-logs Follow service logs in real time
make docker-restart Restart all services
make test Run backend tests
make frontend-test Run frontend tests
make opa-test Run Rego policy tests
make lint Check code style
