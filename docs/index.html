Welcome to CloudLine

CloudLine automatically scans your Amazon Web Services (AWS) account and finds security problems before hackers do. Think of it as a security guard that watches your cloud 24/7.

What Is CloudLine?

Simple Analogy Imagine your AWS account is a house. CloudLine walks through every room, checks if the doors are locked, windows are closed, the alarm system is on, and the smoke detectors have batteries. If something is wrong, it tells you exactly what to fix and how.

CloudLine is a security scanning tool built for Amazon Web Services (AWS). It checks 21 different security rules across your cloud setup and tells you:

  • What is wrong — e.g., "Your S3 bucket is publicly accessible"
  • How serious it is — Critical, High, or Medium severity
  • How to fix it — step-by-step instructions, CLI commands, and Terraform code
  • Which regulations it affects — CIS, NIST, PCI DSS, HIPAA, SOC 2

What AWS Services Does It Check?

CloudLine scans these 8 AWS services for security misconfigurations:

AWS Service What It Does What CloudLine Checks
IAM Manages who can access your AWS Root MFA, password policy, user MFA, unused credentials, Access Analyzer
S3 Stores files in the cloud Public access blocking, encryption
EC2 Runs virtual servers Security groups, IMDSv2, EBS encryption
VPC Your private network in AWS Flow logs, Network ACL rules
RDS Managed databases Public access, encryption, backups
Lambda Runs code without servers Tracing, environment encryption
CloudTrail Logs all API activity Multi-region, log validation, CloudWatch alarms
GuardDuty Threat detection service Enabled/disabled status

The 5 Security Domains

CloudLine organizes its 21 security checks into 5 categories (called "domains"):

1. Identity & Access Control

5 checks — Makes sure only the right people can access your AWS account and that their access is secure.

2. Data Protection

6 checks — Ensures your data is encrypted, backed up, and not publicly exposed.

3. Network Security

3 checks — Verifies your network is locked down and not open to attackers.

4. Logging & Monitoring

4 checks — Confirms that AWS is recording activity so you can detect suspicious behavior.

5. Threat Detection

2 checks + 1 cross-resource — Checks that AWS security services and serverless functions are properly configured.
