How CloudLine Works

CloudLine uses a 6-layer system to find, score, and fix security problems in your AWS account. Here is how each layer works, explained simply.

Think of It Like a Hospital Layer 1 collects your health data (blood tests, X-rays). Layer 2 is the doctor who reads the results. Layer 3 monitors you 24/7. Layer 4 tells you how sick you are. Layer 5 gives you the medicine. Layer 6 is the patient portal where you see everything.
📦

Layer 1: Data Collection

What it does: Gathers information about your AWS resources.

CloudLine has 8 "collectors" — small programs that talk to AWS and ask questions like:

  • "What S3 buckets exist and are they publicly accessible?"
  • "Do all IAM users have MFA turned on?"
  • "Are EC2 instances using the secure metadata service?"

Each collector retrieves data from one AWS service and converts it into a standard format (JSON). This means all 8 services produce data that looks the same, making it easy for the next layer to process.

💡
The 8 collectors: IAM, S3, EC2, VPC, RDS, Lambda, Logging (CloudTrail/Config/CloudWatch/ GuardDuty), and KMS (Secrets Manager/Backup).
🔎

Layer 2: Policy Engine (OPA/Rego)

What it does: Analyzes the collected data against 21 security rules.

Simple Analogy Think of this like a checklist. The policy engine goes through each item and checks "Is this secure? Yes or No." If the answer is No, it creates a "violation" with details about what is wrong.

CloudLine uses Open Policy Agent (OPA) — an open-source tool used by companies worldwide to enforce security rules. The rules are written in a language called Rego.

Each rule checks one specific thing. For example, Check #04 asks: "Does this S3 bucket block all public access?" If the answer is no, it creates a violation that says:

Result "check_id": "CHECK_04" "severity": "critical" "reason": "S3 bucket does not block public access" "resource": "arn:aws:s3:::my-bucket"
⚡

Layer 3: Real-Time Detection

What it does: Watches for changes in your AWS account and checks them instantly.

Instead of only scanning once a day, CloudLine also monitors your AWS account in real-time. Here is how:

Someone changes something in AWS

For example, someone opens SSH port 22 on a security group.

AWS CloudTrail records the change

CloudTrail logs every API call made in your account.

EventBridge detects the event

AWS EventBridge matches the event name (e.g., "AuthorizeSecurityGroupIngress") against CloudLine's list of 20 tracked events.

Lambda function triggers

A serverless function runs automatically, re-collects the resource data, and re-evaluates the relevant policy.

Alert sent if there is a problem

If the change introduced a security issue, CloudLine sends an alert via SNS and updates the dashboard in real-time.

⚠️
CloudLine tracks 20 specific CloudTrail events. Not every AWS action triggers a re-scan — only the ones that could affect security.
📈

Layer 4: Risk Scoring

What it does: Calculates how dangerous each finding is on a scale of 0 to 100.

Not all security problems are equally dangerous. A publicly accessible database (score: 95) is far worse than a missing CloudWatch alarm (score: 35). CloudLine uses 5 factors to calculate risk:

Factor Weight What It Measures
Severity 30% How serious is the vulnerability? (Critical, High, Medium)
Exploitability 25% How easy is it for an attacker to exploit this?
Blast Radius 20% How much damage could this cause if exploited?
Data Sensitivity 15% Does the affected resource handle sensitive data?
Compliance Impact 10% How many compliance frameworks does this violate?

Risk categories:

  • Critical 90–100 — Fix immediately
  • High 70–89 — Fix within 24 hours
  • Medium 40–69 — Fix within 1 week
  • Low 0–39 — Fix when convenient
🔨

Layer 5: Automated Remediation

What it does: Helps you fix the problems — from simple instructions to fully automatic fixes.

CloudLine offers 3 levels ("tiers") of help:

Tier 1 Suggestions

Every finding comes with step-by-step console instructions, an AWS CLI command, and a Terraform snippet. You follow the instructions manually.

Available for all 21 checks.

Tier 2 One-Click Fixes

For 6 common problems, you can click a button and CloudLine fixes it for you automatically. You can also undo the fix if needed.

Available for: S3 public access, CloudTrail, security groups, IMDSv2, EBS encryption, password policy.

Tier 3 Auto-Remediation

When enabled, CloudLine automatically fixes new violations as they are detected. You configure which checks should be auto-remediated.

Optional — you control which checks run automatically.

💻

Layer 6: Compliance Dashboard

What it does: Shows you everything in a visual web interface.

The dashboard is a web application built with React that shows:

  • Overall compliance score — a percentage showing how secure your AWS account is
  • Violations list — sortable and filterable by severity, domain, or status
  • Trend charts — are things getting better or worse over time?
  • Risk summary — highest-risk findings that need immediate attention
  • Real-time alerts — instant notifications when new violations are detected
  • Remediation panel — fix issues directly from the dashboard
